# All-Round-TV AI 协作规范

## 适用范围

本文件适用于整个仓库。进入 `apps/web` 或 `services/api` 工作时，还必须遵循对应目录中的 `AGENTS.md`。

## 项目目标

V2 是私有部署的多用户影视聚合系统。前端使用 Vben/Vue 3，后端使用 FastAPI，PostgreSQL 是业务事实源，Redis 用于可重建的会话、缓存和任务状态。

## 开始工作前

1. 阅读 `docs/PROJECT_REFACTORING.md`。
2. 阅读相关的架构、开发方案和最近目录的 `AGENTS.md`。
3. 检查工作区状态，保护用户已有修改。
4. 跨模块功能必须先在 `docs/plans/active/` 创建或更新方案。

## 目录边界

- 前端业务只能进入 `apps/web`。
- 可复用前端包进入 `packages`。
- 后端业务只能进入 `services/api`。
- 部署配置进入 `infra`。
- 跨系统测试进入根 `tests`。
- 长期有效的说明进入 `docs`，不得散落在源码目录。
- `legacy/v1` 默认只读，不得格式化、升级依赖或顺手修复。

## 安全红线

- 不读取、提交、记录或输出真实密钥、令牌和用户隐私。
- 不得实现任意 URL 代理，不得绕过 SSRF、权限、CSRF 或限流保护。
- 前端不得直接访问豆瓣、TMDB、CMS 或媒体上游。
- 日志不得包含密码、Cookie、邀请令牌、完整媒体 URL 或敏感响应正文。
- 数据库模型变化必须包含 Alembic 迁移、回滚考虑和测试。

## 工程要求

- 保持 Router、Service、Repository、Provider 的职责边界。
- OpenAPI 是前后端接口事实源；生成客户端不得手工修改。
- 新依赖必须说明用途，避免重复能力和无维护依赖。
- 修改行为时同步修改测试和文档。
- 完成任务前运行受影响范围的格式、类型、测试和构建检查。

## 过程记录

- 每个自然日只维护一份 `docs/worklogs/YYYY/YYYY-MM-DD.md`。
- 工作记录包含目标、完成项、决策、验证、问题和下一步，不记录命令流水账。
- 可复用的 AI 指令、失败经验和评测结论必须脱敏后保存到 `docs/ai/`。
- 原型以 Figma 为事实源，确认版本导出到 `docs/prototypes/`。

## 完成定义

- 代码、测试、文档和迁移保持一致。
- 所有执行过的验证及结果写入当日工作记录。
- 未解决风险明确记录，不以静默降级代替说明。
- 不修改任务范围之外的用户文件。

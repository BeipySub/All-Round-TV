# V2 核心用户流程

> 版本：v0.1 草案  
> 日期：2026-07-13  
> 输入：[产品需求](../../product/product-requirements.md)、[状态机](../../product/state-machines.md)

## 流程一：管理员邀请用户

```mermaid
flowchart LR
  A[管理员进入邀请页] --> B[创建一次性邀请]
  B --> C[复制站内邀请链接]
  C --> D{用户打开链接}
  D -->|有效| E[设置用户名、显示名和密码]
  D -->|过期/撤销/已使用| F[说明状态并提示联系管理员]
  E --> G[阅读并接受当前声明]
  G --> H[账户创建并原子消费邀请]
  H --> I[进入首页]
```

关键规则：邀请链接只显示一次完整值；管理员列表只显示摘要和状态；提交开户时若邀请刚被消费，用户停留在结果页而非重复创建。

## 流程二：登录与访问恢复

```mermaid
flowchart LR
  A[访问受保护站内地址] --> B{已有有效会话?}
  B -->|否| C[登录]
  C -->|成功| D{声明为当前版本?}
  B -->|是| D
  D -->|否| E[确认声明]
  D -->|是| F[返回原站内目标]
  E -->|接受| F
  E -->|拒绝| G[退出]
```

播放地址恢复到对应剧集页并重新创建播放会话，不恢复旧播放令牌。

## 流程三：发现、搜索到播放

```mermaid
flowchart LR
  A[首页推荐/继续观看] --> D[影视详情]
  B[搜索页提交查询] --> C{聚合结果}
  C -->|有结果/部分成功| D
  C -->|空| E[调整关键词或来源]
  C -->|全失败| F[重试或查看来源状态]
  D --> G[选择来源和剧集]
  G --> H[创建播放会话]
  H -->|成功| I[播放器]
  H -->|失败| J[解释原因并重试/换源]
```

搜索进行中发起新查询时取消旧任务；部分来源失败不遮挡成功结果。详情主按钮按“继续播放/开始播放”变化。

## 流程四：播放失败与手动换源

```mermaid
flowchart LR
  A[播放中] --> B{可自动恢复?}
  B -->|是| C[缓冲/重试当前来源]
  C -->|恢复| A
  C -->|超过阈值| D[终止错误]
  B -->|否| D
  D --> E[展示错误阶段和候选来源]
  E --> F{用户选择}
  F -->|重试当前源| G[创建新会话]
  F -->|确认候选源与目标集| H[创建候选源会话]
  F -->|返回详情| I[保留当前标准剧集]
  G --> A
  H --> A
```

任何候选源都不能自动切换；无法精确匹配季/集时必须让用户确认目标集。

## 流程五：用户提交来源到管理员启用

```mermaid
flowchart LR
  A[用户填写候选来源] --> B[提交]
  B --> C[网络安全检查]
  C -->|失败| D[安全拒绝终态]
  C -->|通过| E[能力探测]
  E -->|失败| F[需要修改]
  E -->|通过| G[等待管理员审核]
  G -->|拒绝| H[显示用户可见原因]
  G -->|需修改| F
  G -->|批准| I[创建已批准未启用来源]
  I --> J[管理员确认启用]
  J --> K[用户可在我的来源中选择]
```

安全检查不可被管理员绕过；修改通过新提交完成，旧记录保留；批准与启用是两个显式动作。

## 流程六：观看进度、收藏与历史

```mermaid
flowchart LR
  A[播放超过10秒] --> B[保存标准剧集进度]
  B --> C{达到完成阈值?}
  C -->|否| D[首页显示继续观看]
  C -->|是| E[标记本集完成]
  D --> F[另一设备继续播放]
  E --> G{自动连播开启且有下一集?}
  G -->|是| H[提示并创建下一集会话]
  G -->|否| I[停留结束态]
```

收藏独立于播放来源；删除历史不删除收藏，继续播放后可重新生成历史。

## 流程七：V1 数据导入

```mermaid
flowchart LR
  A[选择本地JSON] --> B[校验与脱敏解析]
  B -->|无效| C[显示字段级错误]
  B -->|有效| D[预览导入/跳过/失败项]
  D --> E{用户确认?}
  E -->|否| F[取消且不写入]
  E -->|是| G[后台处理]
  G --> H[完成/部分完成/失败报告]
```

旧自定义源只转成候选提交；重复导入不重复创建；预览阶段不写业务数据。

## 流程八：管理员治理

```mermaid
flowchart LR
  A[管理概览] --> B{待处理类型}
  B --> C[邀请与用户]
  B --> D[来源审核与健康]
  B --> E[Provider/系统故障]
  C --> F[创建邀请/禁用/恢复/角色调整]
  D --> G[审核/启停/归档]
  E --> H[查看脱敏详情/重试允许任务]
  F --> I[写入审计]
  G --> I
  H --> I
```

最后一名有效管理员保护必须在禁用和降级确认框中直接说明；审计记录不可修改或删除。

## 跨流程中断规则

- 会话失效：保存可恢复的站内目标，登录后恢复；播放会话重新创建。
- 声明升级：阻断所有内容页面，接受后回到原目标。
- 用户被禁用：终止当前操作并退出，不展示账户是否存在等细节。
- 来源被暂停：禁止新搜索和播放会话，已有播放会话自然到期。
- 对象状态冲突：刷新到服务端最新状态，禁止旧页面覆盖审核或治理结果。

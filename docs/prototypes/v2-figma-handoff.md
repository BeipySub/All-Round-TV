## 3. 组件与变体

- 应用头部、移动底栏、管理侧栏。
- 影视卡片：默认、悬停/焦点、收藏、进度、不可播放、海报缺失。
- 来源状态：健康、降级、暂停、未启用、归档。
- 异步状态：骨架、空、部分成功、错误、无权限、过期。
- 剧集按钮：默认、选中、已看、进行中、不可用、未标准化。
- 审核状态标签和审核操作确认框。
- 播放错误覆盖层、换源候选项和目标剧集确认。
- Toast、Alert、Modal、Drawer 与请求追踪 ID 区域。

## 4. 原型连接

至少建立四条可点击演示：

1. 有效邀请 → 开户 → 声明 → 首页。
2. 搜索部分成功 → 详情 → 播放错误 → 确认换源 → 恢复播放。
3. 用户提交 → 检测完成 → 管理员审核 → 确认启用 → 用户选择来源。
4. V1 上传 → 预览 → 确认 → 部分完成报告。

## 5. 命名与标注

- Frame：`<编号> / <页面> / <状态> / <断点>`。
- Component：`Domain/Component`，Variant 使用 `state`、`size`、`platform`、`status`。
- 流程起点使用 Figma Flow starting point 命名为 `FLOW-01` 等。
- 每个异常 Frame 标注触发条件、允许动作、返回目标和对应 PRD/状态机编号。

## 6. 评审门禁

- 产品：任务是否完成、文案是否可理解、异常是否可恢复。
- 权限：入口、隐藏、403/404、本人数据边界是否一致。
- 安全：是否暴露 URL、token、密钥、上游正文或账户枚举信息。
- 前端：响应式、组件复用、焦点和加载状态是否可实现。
- 后端：状态转换、并发冲突和异步任务反馈是否有事实依据。

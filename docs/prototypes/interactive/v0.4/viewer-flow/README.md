# 观影、推荐与广告追踪原型 v0.4

直接在浏览器打开 `index.html`。本原型不请求网络、不访问真实 Provider/CMS/媒体，也不属于生产前端代码。

## 本轮重新设计

- 发现页改为影视内容导向的主视觉、竖版推荐轨道和横向热播区，不再呈现管理后台式首页。
- 搜索页集中呈现查询、筛选、部分来源失败和聚合结果。
- 详情页明确分离剧集、来源和最终播放组合。
- 播放页采用“视频主区＋同类推荐侧栏”，窄屏时推荐移动到视频下方。
- 同类推荐展示原因；点击先进入影视详情，不直接播放另一部影视。

## 广告追踪演示

### 使用已有可信标记

1. 进入播放器。
2. 使用顶部原型工具栏“进入已标记广告段”。
3. 默认显示“跳过广告”；点击后进度前移并出现撤销操作。
4. 开启“可信广告自动跳过”后再次进入广告段，观察自动跳过和撤销反馈。

### 用户手动标记

1. 点击播放器下方“标记广告开始”。
2. 点击“标记广告结束”。
3. 在确认框调整开始、结束和区段类型。
4. 保存后，该标记先对本人有效，不会立即影响其他用户。

## 规则边界

- 广告区段绑定当前来源、来源条目、剧集和媒体版本。
- 跳过只改变播放器时间，不删除或重写 HLS 分片。
- 自动跳过默认关闭，只处理高置信度或管理员确认的区段。
- 自动跳过和手动跳过均提供撤销路径。
- 片头、片尾和预告不默认等同于广告。
- 同类推荐不会在当前视频结束后自动切换到另一部影视。

## 评审重点

- 新发现页是否更像影视产品而非后台系统。
- 播放页推荐是否有价值但不干扰观看。
- “标记开始→标记结束→确认”是否足够易懂。
- 手动跳过、自动跳过和撤销之间的关系是否清楚。
- 移动端推荐和广告工具是否仍容易操作。

Figma 仍是最终原型事实源；本目录用于在 Figma 可用前验证产品流程和交互。

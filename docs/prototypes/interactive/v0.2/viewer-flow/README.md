# 观影主流程可点击原型 v0.2

直接在浏览器打开 `index.html`。本原型不请求网络、不访问真实 Provider/CMS/媒体，也不属于生产前端代码。

## 演示路径

1. 首页“继续播放”直达播放器；点击内容卡片进入详情。
2. 或进入搜索，提交预填关键词并查看部分成功状态。
3. 在详情分别选择剧集和来源，核对底部播放组合后开始播放。
4. 在播放器顶部原型工具栏点击“触发播放异常”。
5. 查看已保留进度和候选来源，确认手动换源。
6. 观察连接新来源的过渡状态，并从原进度恢复播放。

## v0.2 优化

- 统一首页主按钮语义，“继续播放”不再绕到详情。
- 将详情页来源、线路、剧集选择改为明确的分步层级。
- 新增最终播放组合和粘性播放操作区，避免用户忘记当前选择。
- 将故障触发器移到原型工具栏，不污染真实播放器控件。
- 换源前明确目标来源、剧集和保留进度，确认后展示连接状态。
- 搜索完成后把键盘焦点移动到主要匹配结果。

## 评审重点

- 首页“继续播放”直达与“查看详情”分流是否符合预期。
- 搜索来源状态是否清楚但不过度干扰。
- 详情的来源、线路、剧集层级是否易理解。
- 错误、重试、人工换源及进度恢复是否足够清楚。
- 390 px 移动布局是否仍能完成主流程。

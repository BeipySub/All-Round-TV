# V2 原型交互规格

## 1. 全局行为

- 桌面导航保持可见；移动端观影区使用底部导航，输入法弹出时自动隐藏。
- 主导航搜索入口只显示图标；点击后打开居中悬浮搜索弹窗并管理焦点，提交后进入完整结果页，Esc、关闭按钮或点击遮罩关闭。
- 站内返回优先恢复列表查询、筛选、滚动位置和展开状态。
- 只有用户明确提交才发起 CMS 搜索；按钮与 Enter 行为一致。
- 保存型操作区分“已保存”“保存失败”“检测中”，禁止用短暂 Toast 隐藏关键结果。
- 删除、清空、禁用、撤销、拒绝、归档等操作必须说明影响；高影响动作需二次确认。

## 2. 搜索

- 提交后搜索框保持可编辑；新查询立即取消旧查询的 UI 订阅。
- 首批结果可用即展示，不等待所有来源完成。
- 来源状态折叠区持续更新，但不能引起结果卡片大幅跳动。
- “加载更多”只推进仍有游标的来源；全部完成后变为结束说明。
- 未选择来源时主操作引导到“我的来源”，返回后保留原查询。

## 3. 详情与播放

- 详情默认恢复上次成功来源和剧集；不可用时预选健康度最高的可用来源并明确说明。
- 点击剧集只更新选中态；点击播放才创建会话，防止浏览剧集时产生媒体请求。
- 播放器控制层使用 ArtPlayer 能力但遵循本规格的权限与错误覆盖层。
- 会话续建最多自动尝试一次；失败后必须给出人工操作。
- 自动连播前显示可取消倒计时；用户最近主动暂停或处于错误态时不自动进入下一集。
- 播放页同类推荐位于视频之外的次级区域，展示推荐原因；点击先到详情，不直接播放其他影视。
- 用户可从播放器标记广告开始/结束，确认前允许微调；有效广告段到达时显示跳过按钮。
- 广告标记仅对已登录且账户有效的注册用户开放；会话失效时引导重新登录，不缓存待提交标记。
- 播放器下方按配置来源展示实际剧集列表；切换来源后刷新剧集，用户选集后再创建播放会话。
- 自动跳过默认关闭，只处理高置信度区段；跳过后提供“撤销并返回广告开始”的短时操作。
- 广告标记和跳过不得表现为修改或删除媒体分片。
- 移动端控制锁锁定手势和控件，保留清晰的长按解锁入口；不提供长按倍速。

## 4. 来源提交与审核

- 提交表单在客户端只做格式提示；服务端安全检查结果才是事实。
- 提交后跳转详情时间线，不让用户重复点击造成重复记录。
- 检测中页面允许离开，列表状态异步更新。
- `needs_revision` 通过“基于此记录新建”创建新版本，不能编辑旧记录。
- 管理员审核前展示检测时间；结果过旧时先重新检测再允许批准。
- 审核动作使用当前版本号防并发覆盖；冲突时刷新并展示谁已处理，不自动重放操作。

## 5. 身份与隐私

- 登录错误统一为账户或凭据不可用，不区分用户名不存在、密码错误或账户禁用。
- 邀请无效页面可以区分过期、撤销和已使用，但不显示被邀请人信息。
- 会话过期弹层只在无法无感恢复时显示；确认后进入登录并保存站内目标。
- 用户清空历史或搜索历史后立即更新页面；失败则恢复原列表并给出明确错误。
- 管理员页面不提供进入其他用户收藏、历史或搜索历史的入口。

## 6. 可访问性与响应式要求

- 所有操作可通过键盘完成，焦点顺序与视觉层级一致。
- 弹窗打开时焦点进入，关闭后返回触发元素；播放器快捷键不抢占表单和弹窗焦点。
- 状态不能只靠颜色表达，必须同时使用文字或图标。
- 正文、按钮和状态文字满足可读对比度；错误摘要允许复制请求追踪 ID，但不复制敏感正文。
- 触控主操作目标建议不小于 44×44 px；移动端不依赖悬停发现功能。

## 7. 原型需验证的问题

1. 详情页“来源—线路—剧集”的层级在移动端是否易于理解。
2. 搜索逐源状态展开是否会干扰结果浏览。
3. 播放错误时，重试与换源的优先级是否清楚。
4. 批准后再启用来源是否能被管理员理解为两个动作。
5. 管理端移动卡片是否足以完成审核而无需桌面表格。

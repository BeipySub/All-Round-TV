# V2 低保真线框

> 本文用于确定区域、层级与操作，不是最终视觉稿。桌面基准 1440 px，移动端基准 390 px；具体栅格由 Figma 评审确定。

## 1. 登录与邀请

```text
┌──────────────────────────────────────────────┐
│                 All-Round-TV                 │
│  ┌────────────────────────────────────────┐  │
│  │ 登录 / 接受邀请                       │  │
│  │ [用户名____________________________]   │  │
│  │ [密码______________________________]   │  │
│  │ [主要操作：登录]                       │  │
│  │ 统一错误 / 邀请状态 / 声明入口         │  │
│  └────────────────────────────────────────┘  │
└──────────────────────────────────────────────┘
```

邀请开户分两步：账户资料 → 声明确认。移动端单列全宽，密码要求在输入时就地反馈。

## 2. 首页

```text
┌ Logo ─ 首页 搜索 收藏 历史 ───────── 管理 头像 ┐
│ [Provider 降级提示，仅异常时出现]              │
│ 继续观看  [卡片+进度] [卡片+进度]              │
│ 我的收藏  [卡片] [卡片]                        │
│ 热门电影                         [换一批]       │
│ [海报] [海报] [海报] [海报] [海报]             │
│ 热门电视剧                       [换一批]       │
└─────────────────────────────────────────────────┘
```

无继续观看或收藏时整个区块不占位；Provider 异常是非阻断横幅，不使用全屏错误。

## 3. 搜索

```text
┌ [搜索关键词________________________] [搜索] ┐
│ 类型▾ 年份▾ 来源▾ □仅可播放                  │
│ 来源状态：成功 4 · 超时 1 · 失败 1 [详情]   │
├─────────────────────────────────────────────┤
│ [海报] 标题 / 年份 / 类型       3 个可用来源 │
│ [海报] 标题 / 年份 / 类型       来源条目     │
│                 [加载更多]                   │
└─────────────────────────────────────────────┘
```

移动端筛选进入底部抽屉；结果先显示，逐源状态可展开。空结果保留查询与筛选，失败态提供重试和来源设置入口。

## 4. 影视详情

```text
┌ [背景/海报] 标题 年份 类型      [收藏]       ┐
│ 简介 · 演职员 · 元数据来源                    │
│ [继续播放 第X集]                              │
├ 来源 A 健康 线路 1 ▾ ────────────────────────┤
│ 剧集：[1] [2] [3] [4] ...       [正/倒序]    │
│ 来源 B 降级 线路 1 ▾                         │
└───────────────────────────────────────────────┘
```

主操作唯一且突出；来源、线路和剧集是逐层选择，不展示上游 URL。移动端主操作可粘底，但不得遮挡剧集列表。

## 5. 播放器

```text
┌ 返回详情 ─ 标题 · S1E3 ───────┬ 同类推荐      ┐
│                               │ 科幻冒险      │
│           视频画面            │ [推荐卡片]    │
│ [广告可跳过 / 缓冲 / 错误层]  │ [推荐卡片]    │
├ 播放 进度(广告标记) 倍速 全屏 ┤ 推荐原因      │
│ [标记开始/结束] [自动跳过开关]│               │
└───────────────────────────────┴───────────────┘
```

终止错误覆盖层包含错误阶段、重试、候选来源和返回详情；候选来源点击后先弹出“来源 + 目标剧集”确认，不立即切换。同类推荐位于视频外侧，点击先进入详情。广告追踪以时间段标记和播放器跳过实现，不删除媒体分片。

## 6. 我的提交

```text
┌ 我的来源提交                         [新建提交] ┐
│ 名称  状态         最近更新  操作              │
│ A源   安全检查中   刚刚      查看              │
│ B源   需要修改     昨天      查看 / 重新提交   │
│ C源   等待审核     2天前     查看 / 撤回       │
└────────────────────────────────────────────────┘
```

详情使用时间线展示提交、检查、探测和审核；地址默认截断，错误为脱敏摘要。

## 7. 管理来源审核

```text
┌ 来源审核 ─ 待审核(4) 需修改 已完成            ┐
│ 列表                 │ 提交详情                │
│ 名称 / 提交人 / 时间 │ 安全检查：通过          │
│ 状态 / 能力摘要      │ 搜索/详情/剧集：通过    │
│                      │ 用户说明                │
│                      │ [需修改] [拒绝] [批准]  │
└────────────────────────────────────────────────┘
```

批准后进入独立“是否启用全局来源”步骤；状态已被其他管理员更新时，禁用操作并刷新详情。

## 8. 管理概览

```text
┌ 侧栏 ─────────┬ 用户 会话 待审 来源健康 Provider ┐
│ 概览          │ [指标卡] [指标卡] [指标卡]       │
│ 用户/邀请     │ 待处理事项                        │
│ 来源/审核     │ [来源审核] [故障来源] [任务失败]  │
│ Provider      │ 系统依赖状态                      │
│ 审计/系统     │ PostgreSQL Redis Worker Storage   │
└───────────────┴───────────────────────────────────┘
```

管理台移动端仍需完成审核主流程；复杂表格转为卡片列表和详情抽屉，不要求移动端同时展示主从双栏。

## 9. 通用页面状态组件

每个数据区块复用以下模式：骨架加载、空状态、部分成功警告、可重试错误、无权限、对象不存在、数据过期。危险确认框必须说明对象、影响和是否可恢复，不使用只有“确定/取消”的模糊文案。

## 1. 页面清单

| 页面 | 入口 | 主要职责 | 关键依赖 |
| --- | --- | --- | --- |
| 首页 | `/`、`index.html`、`/s=<keyword>` | 推荐、搜索、搜索结果、详情弹窗、历史、设置、密码验证、声明 | `app.js`、`ui.js`、`api.js`、`search.js`、`douban.js` |

## 2. 访问与声明

### 2.1 共享访问密码

- 部署端把 `PASSWORD` 的 SHA-256 哈希注入 HTML 的 `window.__ENV__.PASSWORD`。
- 用户在浏览器输入密码，浏览器计算 SHA-256 后与页面中的哈希直接比较。
- 验证状态保存在 `passwordVerified`，有效期为 90 天。
- 未配置密码时，页面阻止核心操作并提示部署者设置环境变量。
- 首页与播放器分别包含密码弹窗。

证据：`js/password.js:7-106`、`index.html:334-353`、`player.html:43-59`。

### 2.2 使用声明

- 用户首次访问首页时显示免责声明。
- 接受状态保存在 `hasSeenDisclaimer`，此后不再展示。
- 声明强调系统仅提供搜索、第三方内容由用户自行判断。

证据：`js/index-page.js:2-18`、`index.html:355-379`。

## 3. 首页发现

### 3.1 豆瓣推荐

- 支持电影与电视剧切换。
- 支持标签筛选、用户自定义标签、删除标签和恢复默认标签。
- 每批展示 16 条，可通过“换一批”递增 `page_start`。
- 通过豆瓣 `search_tags` 与 `search_subjects` 非正式接口获取数据。
- 主代理失败后使用 `api.allorigins.win` 作为备用跨域代理。
- 推荐卡片点击后将标题填入搜索框并触发 CMS 搜索，而不是进入豆瓣详情。

证据：`js/douban.js:1-113`、`js/douban.js:384-499`。

### 3.2 推荐开关

- 用户可关闭豆瓣区域。
- 搜索结果展示时自动隐藏推荐区，返回首页时恢复。

证据：`js/douban.js:57-134`。

## 4. 资源源管理

### 4.1 内置源

- 资源源由 `API_SITES` 与 `customer_site.js` 合并提供。
- 每个源包含 API 地址、显示名称，可选详情页地址和成人内容标记。
- 支持全选、全不选、仅选择普通源。

静态代码中实际存在的源：

| Key | 地址 | 备注 |
| --- | --- | --- |
| `testSource` | `https://www.example.com/api.php/provide/vod` | 空内容测试源，标记成人 |
| `qiqi` | `https://www.qiqidys.com/api.php/provide/vod` | 七七资源 |

首次运行默认选择的 `tyyszy`、`dyttzy`、`bfzy`、`ruyi` 均未出现在当前配置中，属于确定缺陷。

证据：`js/config.js:23-31`、`js/customer_site.js:1-13`、`js/app.js:29-42`。

### 4.2 用户自定义源

- 用户可添加、编辑、删除并选择自定义源。
- 字段包括 `name`、`url`、可选 `detail`、`isAdult`。
- 仅验证地址以 HTTP/HTTPS 开头，不执行可信域名或网络边界限制。
- 自定义源直接进入搜索和详情流程，不需要审核。

证据：`js/app.js:209-469`。

### 4.3 成人源与过滤

- 内置源和自定义源均可标记成人内容。
- 开启“黄色内容过滤”时按 `type_name` 关键词过滤搜索结果。
- 选择成人源后，前端强制关闭并禁用该过滤开关。
- 过滤只发生在结果渲染前，不影响请求、历史或播放。

证据：`js/app.js:112-207`、`js/app.js:721-729`。

## 5. 搜索

- 用户输入关键词并选择一个或多个资源源。
- 每个源先请求第一页，再根据 `pagecount` 并发请求最多 50 页。
- 多个源同时并发，单请求超时 15 秒。
- 失败源静默返回空数组，不阻断其他源。
- 合并结果按影视名称排序，同名时按来源排序。
- URL 更新为 `/s=<keyword>`，支持刷新后自动重新搜索。
- 最近搜索最多保存 5 条，保存 60 天，支持单条删除和全部清空。
- 结果显示封面、标题、类型、年份、备注和来源。

证据：`js/search.js:1-127`、`js/app.js:607-811`、`js/ui.js:144-305`。

已观察到的行为问题：

- `API_CONFIG.search.maxPages=50` 可使一次搜索产生大量并发请求。
- `AGGREGATED_SEARCH_CONFIG.maxResults` 等配置未被主搜索流程使用。
- 搜索结果数量在成人内容过滤前计算，显示数量可能与实际卡片数量不同。
- 主搜索只按源内 ID 区分结果，没有跨源影视归并。

## 6. 详情与剧集

- 点击搜索结果后打开详情模态框。
- 标准 CMS 详情使用 `?ac=videolist&ids=<id>`。
- 从 `vod_play_url` 的第一个 `$$$` 播放组解析剧集，以 `#` 分集、`$` 分名称与地址。
- 特殊源可抓取 HTML 详情页，并用正则提取 M3U8 地址。
- 详情展示类型、年份、地区、导演、演员、备注和简介。
- 剧集支持正序/倒序与复制全部播放链接。

证据：`js/api.js:75-193`、`js/api.js:212-355`、`js/app.js:858-987`。

## 7. 播放

- 详情页选择剧集后先进入 `watch.html`，约 2.8 秒后跳转到 `player.html`。
- 播放器从 URL 和 localStorage 组合恢复标题、剧集、来源和进度。
- ArtPlayer 负责 UI，hls.js 负责 M3U8。
- 支持自动播放、画中画、网页全屏、系统全屏、截图、倍速、AirPlay 和迷你进度条。
- 支持上一集、下一集、自动连播、集数排序和复制当前链接。
- 支持键盘快捷键、移动端长按三倍速、控制栏锁定和精确进度点击。
- HLS 网络错误与媒体错误有有限恢复逻辑。
- 视频开始播放后写入观看历史，并定期保存进度。

证据：`js/app.js:989-1138`、`js/watch.js:1-94`、`js/player.js:117-1004`。

## 8. 换源

- 播放页按当前标题在所有已选源重新搜索。
- 每个源优先选择完全同名结果，否则选择第一条。
- 请求详情后对第一集执行浏览器 HEAD/no-cors 测速。
- 资源按测速结果排序，当前资源置顶。
- 切换时尽量保持相同集数索引。

证据：`js/player.js:1435-1793`。

## 9. 历史与进度

- 观看历史最多 50 条。
- 记录标题、来源、CMS ID、剧集索引、直接地址、播放器地址、完整剧集数组、播放位置、时长和更新时间。
- 同一条目的判定主要依赖标题、来源和 `showIdentifier`。
- 从历史恢复时会尝试重新请求详情并刷新剧集列表，失败则使用缓存剧集。
- 单集进度另存为 `videoProgress_<encoded-url>`。
- 保存触发点包括 5 秒节流、30 秒定时器、暂停、切后台和离开页面。

证据：`js/ui.js:370-765`、`js/player.js:1076-1270`。

## 10. 配置导入导出

- 导出文件名为 `LibreTV-Settings_<timestamp>.json`。
- 格式包含 `name`、`time`、`cfgVer`、`data`、`hash`。
- 导出资源选择、自定义源、过滤开关、豆瓣开关、观看历史和搜索历史。
- 支持本地 JSON 文件导入和远程 URL 导入。
- 导入前使用 SHA-256 检查 `data` 是否与 `hash` 一致。

证据：`js/app.js:1141-1354`。

## 11. 其他能力

- PWA manifest 与 Service Worker 注册，但 Service Worker 明确不缓存任何资源。
- 每页访问时读取本地 `VERSION.txt`，并访问 GitHub 或代理检查上游版本。
- About 页面提供隐私政策、免责声明和投诉说明。
- 搜索路径包含基础 SEO/JSON-LD 信息。

证据：`manifest.json`、`service-worker.js`、`js/version-check.js`、`about.html`。

## 12. 下一阶段产品输入

下一阶段不能简单复制 V1 UI，需要重点决定：

- 推荐内容与 CMS 搜索是两个模型还是一个统一影视条目。
- 搜索是否继续自动拉取所有分页，还是改为分页/渐进加载。
- 换源匹配使用标题模糊匹配还是标准影视 ID 映射。
- 历史记录是否保存直接播放地址，还是保存稳定资源条目与剧集引用。
- 成人内容是否继续支持，以及如何落实角色、部署和合规策略。
- PWA、复制原始播放链接、广告过滤是否保留。

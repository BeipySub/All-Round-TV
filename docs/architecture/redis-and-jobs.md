# Redis 与异步任务设计

## 1. Redis 数据

| 前缀 | 用途 | 可重建/失效行为 |
| --- | --- | --- |
| `session:` | 登录会话 | 丢失后重新登录 |
| `playback:` | 播放会话与允许资源表 | 丢失后重新创建会话 |
| `csrf:` | CSRF 辅助状态 | 随会话重建 |
| `cache:provider:` | 豆瓣/TMDB 缓存 | 回源或明确降级 |
| `cache:search:` | 规范化查询和逐源结果 | 重新搜索 |
| `cache:recommendation:` | 推荐排序结果 | 从 PostgreSQL/Provider 重建 |
| `ratelimit:` | 登录、搜索、媒体、标记限流 | 窗口结束恢复 |
| `job-status:` | 面向 UI 的短期任务进度 | 从 PostgreSQL 业务状态恢复 |

Redis 值不得包含密码、完整上游 URL 或无需存在的第三方响应正文。

## 2. ARQ 任务

- 来源安全检查和能力探测。
- 来源周期健康检查。
- Provider 推荐与元数据缓存刷新。
- 标准影视映射候选计算。
- MediaVersion 指纹计算或刷新。
- 广告标记聚合、争议重算和过期版本失效。
- 搜索历史 90 天清理和用户历史上限整理。
- V1 导入处理和结果汇总。

## 3. 幂等与重试

- 任务参数使用内部 ID，不传任意 URL；Worker 从受控记录读取目标。
- 幂等键包含任务类型、对象 ID 和配置/版本号。
- 网络超时使用有限指数退避；安全拒绝、格式错误和权限错误不重试。
- 任务最终失败写 PostgreSQL 状态或脱敏结果，不能只留在 Redis。
- 多 Worker 聚合广告标记时使用数据库条件更新或互斥任务键，防止重复区段。

## 4. 队列划分

- `security`：来源安全检查，高优先级且严格资源限制。
- `provider`：豆瓣/TMDB/CMS 元数据与搜索相关任务。
- `media`：媒体版本分析与健康检查，不下载完整视频。
- `aggregation`：推荐和广告标记聚合。
- `maintenance`：清理、导入和周期任务。

不同队列设置独立并发和超时，避免媒体任务挤占登录或审核流程。

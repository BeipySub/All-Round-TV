# 2026-07-14 开发记录

## 目标

重新设计本地交互原型，并为播放器增加同类影视推荐及来源内广告时间段追踪、标记和跳过能力。

## 完成项

- 将广告能力从“删除分片式过滤”重新定义为媒体版本级时间段追踪。
- 明确用户个人标记、多人聚合、系统检测、管理员确认和争议处理状态。
- 明确手动跳过、可信自动跳过、撤销和错误报告边界。
- 定义播放页同类型/同分类推荐逻辑，推荐点击先进入详情。
- 重做 v0.4 发现、搜索、详情和播放页面视觉与信息层级。
- 在播放原型中加入同类推荐、广告段提示、自动跳过开关、撤销和用户标记流程。
- 迭代 v0.5：将搜索改为图标触发的下滑面板，在播放器下方增加按配置来源选集，并限制广告标记为有效注册用户。
- 迭代 v0.6：将下滑搜索面板改为居中悬浮模态框，支持焦点管理、Esc、关闭按钮、遮罩关闭和最近搜索填充。
- 用户同意以 v0.5 作为下一阶段交互基线；保留最终视觉和 Figma 导出待办。
- 建立技术设计方案，完成系统边界、领域模型、API 草案、播放/推荐/广告追踪及 Redis/ARQ 设计。

## 决策

- 自动跳过默认关闭，只作用于高置信度或管理员确认区段。
- 用户本人标记先仅对本人有效，不能立即影响其他用户。
- 访客、会话失效或禁用账户不能创建或提交广告标记。
- 广告标记绑定来源、来源条目、标准剧集和媒体版本指纹，不跨版本直接复用。
- 跳过只调整播放时间，不修改 HLS 清单或媒体分片。
- 推荐不自动播放另一部影视，用户先进入详情确认。

## 目标

依据 V1→V2 功能矩阵，逐项确定 V2 的产品逻辑、角色权限、信息架构和核心业务状态机。

## 完成项

- 建立业务规则、角色权限与数据可见性。
- 建立认证、观影、播放器和管理页面树。
- 定义邀请、用户、会话、声明、搜索、来源、播放、导入和 Provider 状态机。
- 将 V1 功能逐项映射到 V2 入口、角色、规则和状态，清除待确认项。
- 建立邀请、登录恢复、发现播放、手动换源、来源审核、进度、V1 导入和管理员治理用户流程。
- 完成桌面与移动端关键页面的低保真线框和交互规格。
- 建立 Figma 文件结构、必画 Frame、组件变体、可点击流程和评审门禁清单。
- 完成首个本地可点击观影原型：首页/搜索、详情、播放器错误和人工确认换源，支持桌面与移动响应式。
- 根据交互反馈迭代至 v0.2：继续播放直达、选集选源分层、播放组合确认、独立故障触发、保留进度换源和连接过渡。
- 继续迭代至 v0.3：增加页面转场、内容错峰进入、按钮反馈、搜索状态、选集选源、播放错误、换源连接和恢复动画，并支持减少动态效果。

## 决策

- 首版不包含成人专用内容/来源、PWA、广告过滤和移动端长按倍速。
- 搜索历史默认跨设备同步，最多 50 条、保留 90 天，可关闭和清理。
- 保留移动端播放器控制锁；推荐标签首版仅由系统提供。
- 播放失败不得静默自动换源，必须由用户确认候选来源。
- 用户导出覆盖本人主要业务数据，但排除秘密、令牌和媒体 URL。

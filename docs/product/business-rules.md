# V2 业务规则

> 本文是产品行为的统一约束；接口、原型和测试不得各自解释。

## 1. 全局规则

| 编号 | 规则 |
| --- | --- |
| BR-GLOBAL-001 | 除登录、邀请开户和使用声明外，所有能力仅对已登录、未禁用且已接受当前声明的用户开放。 |
| BR-GLOBAL-002 | PostgreSQL 是业务事实源；Redis 状态丢失只能导致会话、缓存或任务重建，不得丢失收藏、历史、审核结论。 |
| BR-GLOBAL-003 | 标准影视条目与 CMS 来源条目分离；收藏、历史和进度绑定标准条目与标准剧集，不绑定媒体 URL。 |
| BR-GLOBAL-004 | 豆瓣、TMDB 或单个 CMS 的失败按来源降级，不能伪装为空结果，也不能阻断无关功能。 |
| BR-GLOBAL-005 | 面向用户和日志的错误必须脱敏，不包含密钥、令牌、Cookie、完整媒体 URL、堆栈或上游正文。 |
| BR-GLOBAL-006 | 首版不接纳成人专用内容或来源，不实现成人关键词过滤、PWA、移动端长按倍速或删除 HLS 分片式广告过滤。 |

## 2. 身份与权限

| 编号 | 规则 |
| --- | --- |
| BR-AUTH-001 | 系统仅支持管理员邀请开户，不开放注册；用户名大小写不敏感且实例内唯一。 |
| BR-AUTH-002 | 邀请一次有效，接受、过期或撤销后不可恢复；需要开户时创建新邀请。 |
| BR-AUTH-003 | 最后一个有效管理员不得被禁用或降级，系统始终保留至少一名有效管理员。 |
| BR-AUTH-004 | 禁用用户立即撤销其全部登录会话和未过期播放会话。 |
| BR-AUTH-005 | 声明升级后，用户必须重新接受才能进入内容区域；接受记录按用户和版本保存。 |
| BR-AUTH-006 | 访问他人私有对象时默认返回“未找到”，避免通过响应差异枚举对象；明确的角色不足返回“无权限”。 |

## 3. 内容、搜索与来源选择

| 编号 | 规则 |
| --- | --- |
| BR-CONTENT-001 | Provider 映射明确时直接合并；否则仅在规范化标题、年份和类型均足够一致时合并，疑似条目保持独立。 |
| BR-CONTENT-002 | 用户实际可用来源是“全局已启用来源”与“用户已选择来源”的交集。 |
| BR-SEARCH-001 | 搜索必须由用户提交，查询为去除首尾空白后的 1–100 个字符；输入过程不触发 CMS 请求。 |
| BR-SEARCH-002 | 初次搜索每源最多三页；继续加载使用各源独立游标，不重新抓取已完成页面。 |
| BR-SEARCH-003 | 每个来源必须返回成功、超时、失败或跳过状态；新查询取消旧查询，旧结果不得覆盖新结果。 |
| BR-SEARCH-004 | 搜索历史默认跨设备同步，最多 50 条、保留 90 天；可停止记录、删除单条或确认后清空。 |
| BR-SOURCE-001 | 普通用户只能提交候选来源，不能直接使其可用于搜索或播放。 |
| BR-SOURCE-002 | 网络安全检查先于能力探测；安全检查失败不可由管理界面绕过。 |
| BR-SOURCE-003 | 审核批准只创建“已批准未启用”全局来源，管理员再次确认后才对用户可选。 |

## 4. 播放、进度与换源

| 编号 | 规则 |
| --- | --- |
| BR-PLAY-001 | 播放会话只能从已审核来源条目和剧集创建，不接受客户端传入的任意 URL。 |
| BR-PLAY-002 | 播放超过 10 秒才形成有效进度；达到总时长 90% 或剩余不超过 2 分钟即完成。 |
| BR-PLAY-003 | 同一标准剧集换源时沿用进度；无法确认剧集等价时必须由用户选择，不回退到第 1 集。 |
| BR-PLAY-004 | 播放失败可展示推荐候选源，但未经用户确认不得切换来源。 |
| BR-PLAY-005 | 分享仅生成需登录的站内影视或剧集地址，不暴露播放令牌和上游地址。 |
| BR-PLAY-006 | 全局来源暂停后不得创建新播放会话；已有短期会话自然到期，不无限续期。 |
| BR-PLAY-007 | 播放页同类推荐必须基于标准影视元数据并排除当前条目；点击推荐先到详情，不自动播放另一部影视。 |
| BR-PLAY-008 | 广告时间段必须绑定来源条目、剧集和媒体版本；跨版本不得凭标题直接复用。 |
| BR-PLAY-009 | 用户本人广告标记可立即用于本人手动跳过；自动跳过只接受高置信度或管理员确认标记且默认关闭。 |
| BR-PLAY-010 | 广告跳过只调整播放时间，必须可撤销，不得修改 HLS 清单、删除分片或把 discontinuity 等同于广告。 |
| BR-PLAY-011 | 只有已登录、账户有效并接受当前声明的注册用户可提交广告标记；会话失效或账户禁用时禁止创建与提交。 |
| BR-PLAY-012 | 播放页选集按当前配置来源分别加载，来源切换不得复用未经标准剧集映射确认的集数索引。 |

## 5. 用户数据与生命周期

| 编号 | 规则 |
| --- | --- |
| BR-DATA-001 | 收藏对象是标准影视条目；同一用户与条目只能存在一条收藏。 |
| BR-DATA-002 | 历史按标准影视聚合展示，内部保留逐集进度；每用户最多 500 个影视条目且不按时间自动过期。 |
| BR-DATA-003 | 清空历史、搜索历史等批量不可逆用户操作必须二次确认。 |
| BR-DATA-004 | V1 导入必须先预览后确认、字段白名单且幂等；旧自定义源只能转为候选提交。 |
| BR-DATA-005 | V2 导出包含本人资料、偏好、来源选择、收藏、历史、进度、搜索历史和本人提交，不含任何秘密或媒体 URL。 |
| BR-DATA-006 | 首版不删除用户主体；禁用代替删除，以保留审计和数据归属。 |

## 6. 冲突处理优先级

发生规则冲突时依次采用：安全与权限约束、管理员全局状态、用户显式选择、用户持久偏好、系统推荐默认值。任何静默降级都不能越过更高优先级规则。

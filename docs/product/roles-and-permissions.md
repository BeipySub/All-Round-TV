# V2 角色与权限

## 1. 主体

- **访客**：仅能登录、使用有效邀请开户、查看并接受声明。
- **用户**：使用影视发现、搜索、播放和个人资料能力，只管理自己的数据与提交。
- **管理员**：继承用户能力，并治理用户、邀请、全局来源、Provider、系统状态和审计。
- **系统任务**：非登录角色，仅按服务身份执行探测、聚合、清理与异步任务，不能绕过安全策略。

首版不设置编辑、审核员、只读管理员等细分角色。

## 3. 数据可见性

| 数据 | 用户 | 管理员 |
| --- | --- | --- |
| 用户资料 | 仅本人完整可编辑字段 | 必要账户字段、状态、角色、创建和最近登录时间 |
| 收藏、历史、搜索历史、偏好 | 仅本人 | 默认不可查看内容明细 |
| 来源提交 | 本人的输入、检测摘要和审核结论 | 全部提交及脱敏检测详情 |
| 全局来源 | 可见名称、健康摘要与能力 | 配置、检测历史和治理操作；敏感凭据仍不可回显 |
| 广告标记 | 本人标记、有效区段、置信度等级 | 聚合与争议摘要；不显示其他用户观看记录 |
| 会话 | 本人可退出当前会话 | 仅会话摘要并可整体撤销，不显示 Cookie/token |
| 审计 | 不可见 | 只读、脱敏；不包含密码、令牌和完整媒体地址 |

管理员身份不等于可以浏览其他用户的观看隐私。若未来确有支持需求，必须新增有目的、可审计且最小化的数据访问流程。

## 4. 授权与响应规则

- 未登录访问受保护页面：跳转登录并保留站内目标。
- 已登录但声明过期：跳转声明页。
- 角色不足访问管理功能：展示 403 页面并记录安全事件摘要。
- 访问不存在或不属于本人的私有对象：统一表现为 404。
- 用户被禁用：当前请求失败、全部会话撤销，后续登录使用统一失败提示。
- 前端隐藏无权限入口只是体验处理；后端必须对每个动作独立鉴权。

# V2 信息架构

## 1. 导航模型

观影前台采用内容型布局，管理台采用 Vben 管理布局，播放器采用最小化独立布局。三者共享身份和设计令牌，但不把管理仪表盘外壳套在观影页面上。

- 桌面主导航：首页、收藏、历史；搜索使用独立图标入口；用户菜单包含来源提交和设置，管理员额外显示管理入口。
- 主导航不显示“搜索”文字项；点击搜索图标后打开居中的全局搜索悬浮弹窗，提交后进入完整搜索结果页。
- 移动端底部导航：首页、搜索、收藏、历史、更多。
- 播放器仅保留返回详情、剧集、来源、播放控制和错误恢复入口。

## 2. 页面树

```text
认证区
├─ /auth/login                         登录
├─ /auth/invite/:token                 邀请开户
└─ /auth/terms                         使用声明
观影区
├─ /                                   首页
├─ /search                             聚合搜索
├─ /media/:mediaId                     标准影视详情
├─ /watch/:mediaId/:episodeId          播放器
├─ /favorites                          收藏
├─ /history                            观看历史
├─ /sources                            我的来源选择
├─ /submissions                        我的来源提交
└─ /settings
   ├─ /profile                         个人资料
   ├─ /playback                        播放偏好
   ├─ /content                         内容与搜索偏好
   ├─ /privacy                         历史与隐私
   └─ /data                            导入与导出
管理区
├─ /admin                              概览
├─ /admin/users                        用户
├─ /admin/invitations                  邀请
├─ /admin/sources                      全局来源
├─ /admin/source-submissions           来源审核
├─ /admin/providers                    豆瓣/TMDB Provider
├─ /admin/audit                        审计
└─ /admin/system                       系统与任务状态
```

## 3. 页面职责

| 页面 | 核心信息 | 主操作 |
| --- | --- | --- |
| 首页 | 继续观看、收藏、热门电影/电视、Provider 降级提示 | 继续播放、查看详情、换一批 |
| 搜索 | 查询、筛选、聚合结果、各源状态 | 搜索、加载更多、打开详情 |
| 详情 | 标准元数据、收藏状态、来源与剧集 | 开始/继续播放、选集、选源、收藏 |
| 播放器 | 当前剧集、进度、按配置来源提供的剧集、广告区段、恢复状态和同类推荐 | 播放控制、按来源选集、确认换源、注册用户标记/跳过广告、查看推荐详情 |
| 收藏/历史 | 本人标准影视列表及进度 | 继续播放、移除、清空历史 |
| 我的来源 | 全局可选来源、健康状态和本人选择 | 启用/停用选择、前往提交 |
| 我的提交 | 本人提交、检测摘要和审核结论 | 新建、撤回、基于失败项重新提交 |
| 设置 | 资料、播放、内容、隐私和数据工具 | 保存偏好、清理数据、导入导出 |
| 管理概览 | 用户、会话、待审、来源、Provider、任务摘要 | 进入待处理列表 |
| 用户/邀请 | 账户与邀请生命周期 | 邀请、撤销、禁用、恢复、角色调整 |
| 来源/审核 | 全局来源与候选提交 | 审核、启停、恢复、归档 |
| Provider/系统 | 配置健康、依赖和任务状态 | 重试允许的任务、查看脱敏详情 |
| 审计 | 不可修改的管理与安全事件 | 筛选、查看摘要 |

## 4. 页面状态规范

所有异步页面按适用范围设计：初始加载、骨架或进度、空状态、成功、部分成功、可重试失败、无权限、对象不存在和数据已过期。搜索必须额外展示逐源状态；播放器必须展示会话创建、缓冲、恢复、会话过期、终止错误、广告段待确认/可跳过/已跳过/撤销和推荐降级；审核页面必须防止旧版本覆盖新状态。

## 5. 原型优先顺序

1. 邀请开户、登录、声明阻断。
2. 首页 → 搜索 → 详情 → 播放 → 手动换源主链路。
3. 用户提交 → 自动检测 → 管理员审核 → 启用来源。
4. 收藏、历史、偏好和数据迁移。
5. 用户治理、Provider 降级、系统状态和审计。

Figma 为事实源；每个确认版本需把关键正常态和异常态导出到 `docs/prototypes/`。

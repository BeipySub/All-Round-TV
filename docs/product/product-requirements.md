# All-Round-TV V2 产品需求

> 状态：产品定义稿  
> 目标：私有部署、多用户、可治理的影视聚合与播放系统  
> 输入：[V1→V2 功能矩阵](../audits/v1-to-v2-feature-matrix.md)

## 1. 产品定位

All-Round-TV V2 帮助一个受信任的小型用户群，在管理员治理的资源源范围内发现、搜索和播放影视内容，并在多个设备间同步收藏、观看历史和播放进度。

产品不托管媒体文件，不允许普通用户把系统当作任意网络代理，也不提供公开注册或匿名内容浏览。

## 2. 用户与核心任务

### 2.1 普通用户

核心任务：

1. 接受管理员邀请并建立账号。
2. 从首页推荐或搜索中找到影视条目。
3. 查看统一详情和可用资源。
4. 从上次进度继续播放、换集或手动换源。
5. 管理收藏、历史、偏好和自己的资源提交。

### 2.2 管理员

管理员同时拥有普通用户能力，额外负责：

1. 邀请、禁用和恢复用户。
2. 审核候选资源源并控制全局可用性。
3. 查看资源、Provider、队列和系统健康状态。
4. 查看不可修改的安全与管理审计记录。

## 3. 首版范围决策

### 3.1 明确保留或新增

- 管理员邀请制和独立用户账号。
- 豆瓣 + TMDB 元数据与推荐，支持单 Provider 降级。
- 管理员审核后的 CMS 多源聚合搜索。
- 标准影视详情、来源列表、剧集与播放。
- ArtPlayer + hls.js 播放体验。
- 上一集、下一集、自动连播、集数排序和键盘快捷键。
- 移动端播放器控制锁。
- 收藏、观看历史、逐集进度与跨设备同步。
- 搜索历史跨设备同步，用户可以关闭记录并清空。
- 用户提交资源源、系统检测和管理员审核。
- 用户数据导出与 V1 JSON 导入。
- 管理端用户、邀请、资源、Provider、审计和系统状态。

### 3.2 明确不进入首版

- 匿名浏览、公开注册和找回密码邮件。
- 成人专用内容分类、成人源和成人关键词过滤。
- PWA 安装与离线能力。
- 移动端长按倍速。
- 原始媒体直链复制。
- 播放失败后未经确认自动换源。
- 自定义豆瓣推荐标签。
- URL 远程导入。
- 直接修改或删除 HLS 分片的广告过滤；广告能力改用时间段追踪与播放端跳过。
- 付费、评论、社交、通知和公开开发者 API。

## 4. 身份与进入系统

### PRD-AUTH-001 首个管理员

- 空系统首次启动时必须完成管理员初始化。
- 初始化完成后，环境变量中的初始密码不再用于日常登录或重建管理员。
- 系统不得存在没有管理员的可用状态；最后一个有效管理员不能被禁用或降级。

### PRD-AUTH-002 邀请开户

- 只有管理员可以创建邀请。
- 邀请链接一次有效、带明确过期时间，可在使用前撤销。
- 用户打开有效邀请后设置用户名、显示名称、密码并接受当前版本使用声明。
- 用户名在实例内唯一且大小写不敏感。
- 邀请过期、已使用或已撤销时必须展示不同状态及联系管理员提示。

### PRD-AUTH-003 登录与会话

- 用户名和密码登录，不提供公开注册。
- 登录失败使用统一提示，不暴露账号是否存在或被禁用。
- 用户可以退出当前会话；管理员禁用用户时撤销其全部会话。
- 会话失效时保留当前站内目标，重新登录后返回原目标；播放会话不直接恢复，需重新创建。

### PRD-AUTH-004 使用声明

- 声明按版本管理。
- 新用户开户时必须接受当前版本。
- 声明版本更新后，用户下次进入内容区前必须重新确认。
- 拒绝声明只能退出系统，不能进入首页、搜索或播放。

## 5. 首页与内容发现

### PRD-HOME-001 首页结构

登录后首页按顺序展示：

1. 继续观看：存在未完成进度时显示。
2. 我的收藏：有收藏时显示最近收藏。
3. 热门电影。
4. 热门电视剧。
5. Provider 状态提示：仅在内容明显降级时显示非阻塞提示。

空区块不占位；豆瓣和 TMDB 都不可用时，继续观看、收藏和 CMS 搜索仍可使用。

### PRD-HOME-002 推荐合并

- 豆瓣与 TMDB 输出统一影视条目。
- 有明确 Provider ID 映射时直接合并；否则按标准化标题、年份和类型进行保守匹配。
- 无法可靠合并时保留独立条目，禁止为了减少重复而错误合并。
- 卡片必须显示内容类型、年份、主要标题和海报来源状态。
- 首版只提供系统标签；用户可隐藏推荐区块，但不能创建自定义标签。

### PRD-HOME-003 换一批

- 每个推荐区块独立换一批。
- 切换后不影响其他区块和页面滚动位置。
- 没有更多内容时回到首批，并明确提示已重新开始。

## 6. 搜索

### PRD-SEARCH-001 查询输入

- 查询去除首尾空白，长度为 1–100 个字符。
- 空查询不发起请求。
- 用户必须至少启用一个全局有效资源源；否则引导到资源选择。
- 连续输入不会自动触发 CMS 搜索；用户按 Enter 或点击搜索后执行。

### PRD-SEARCH-002 聚合策略

- 搜索只访问用户已选择且管理员启用的资源源。
- 初次搜索每个源最多读取前三页；用户“加载更多”时继续推进各源游标。
- 单个源失败不阻断其他源；界面显示成功、超时、失败和跳过数量。
- 结果按标准影视条目聚合，同一条目下展示可用来源数量。
- 无法映射到标准影视条目的 CMS 结果仍可展示为“来源条目”，但不能伪造 Provider 元数据。

### PRD-SEARCH-003 排序与筛选

默认排序依次考虑：

1. 标题完全匹配。
2. 标题前缀匹配。
3. 标题包含与别名匹配。
4. 年份、类型一致性。
5. 可用来源数量与来源健康状态。

用户可筛选电影/电视剧、年份、来源和“仅显示可播放”。首版不提供成人内容筛选。

### PRD-SEARCH-004 结果状态

- 至少一个源成功且有结果：成功或部分成功。
- 至少一个源成功但无结果：空结果，可提示部分源失败。
- 所有源失败：失败状态，提供重试和查看源状态入口。
- 用户离开或发起新查询：旧查询取消，旧结果不得覆盖新查询。

### PRD-SEARCH-005 搜索历史

- 默认保存最近 50 条成功提交的查询，跨设备同步，保留 90 天。
- 相同规范化查询只保留最新一条。
- 用户可以关闭后续记录；关闭不会自动删除已有记录。
- 用户可删除单条或清空全部，清空需要确认。

## 7. 详情与可播放资源

### PRD-MEDIA-001 标准详情

- 详情页围绕标准影视条目展示标题、别名、类型、年份、地区、简介、演职员、海报和 Provider 来源。
- 元数据冲突时按字段显示选定值，不向用户混合互相矛盾的信息。
- 管理员后续可修正错误映射；普通用户首版只可查看。

### PRD-MEDIA-002 来源与剧集

- 只展示管理员启用且用户已选择的来源。
- 来源显示健康状态、最近检测时间、播放线路和剧集数量。
- 多个播放线路全部保留，不再只取第一组。
- 剧集使用稳定 ID、季号、集号和显示名称；无法解析季集时保留原始顺序并标注“未标准化”。
- 不向用户展示原始上游 URL。

### PRD-MEDIA-003 主播放操作

- 有未完成记录时主按钮为“继续播放”。
- 没有历史时主按钮为“开始播放”。
- 默认选择上次成功来源；不可用时选择用户启用范围内健康度最高的来源。
- 只有一个可播放剧集时直接进入播放；多集内容沿用上次集数或第 1 集。

## 8. 播放器

### PRD-PLAY-001 创建播放会话

- 用户从标准影视、来源条目和剧集创建短期播放会话。
- 会话只允许访问服务端已解析的媒体资源，不接受用户提供任意 URL。
- 会话失效但用户仍登录时，播放器可以请求重新创建当前剧集会话；失败时进入明确错误状态。

### PRD-PLAY-002 播放控制

- 支持播放/暂停、进度、音量、静音、倍速、画中画、网页全屏和系统全屏。
- 支持上一集、下一集、自动连播、集数正倒序和移动端控制锁。
- 快捷键在输入框、弹窗和辅助技术焦点中不得误触发。
- 首版不提供长按倍速；广告处理采用时间段标记和播放器跳过，不修改 HLS 清单或分片。

### PRD-PLAY-003 进度与完成

- 播放超过 10 秒才创建有效进度。
- 进度达到总时长 90%，或剩余不足 2 分钟时标记该集已完成。
- 已完成内容默认从头播放，但用户可从历史选择继续。
- 进度按用户 + 标准影视 + 剧集保存，不依赖来源 URL。
- 切换来源但仍是同一标准剧集时沿用进度。

### PRD-PLAY-004 播放失败与换源

- 失败时显示失败阶段、可理解原因、重试和候选来源。
- 系统可以预选推荐来源，但必须由用户确认后切换。
- 换源优先匹配相同季号和集号；无法精确匹配时不得自动回到第 1 集。
- 用户确认降级匹配后才可切换，并明确目标集数。
- 错误与用户反馈进入聚合健康指标，不记录完整媒体 URL。

### PRD-PLAY-005 分享

- 只能复制站内影视或剧集链接。
- 分享链接要求接收者登录并拥有该来源访问权。
- 不复制播放会话 token 或上游媒体地址。

### PRD-PLAY-006 同类影视推荐

- 播放页在不遮挡视频的次级区域展示同类型、同分类和相近标签的标准影视条目。
- 推荐优先使用当前影视的类型、分类、地区、年份和 Provider 标签，并排除当前影视、用户明确隐藏的条目及不可播放条目。
- 推荐卡片必须解释主要依据，例如“同为科幻冒险”或“同类型热门”。
- 点击推荐先进入标准影视详情，不直接创建播放会话；本集结束也不得自动跳转到另一部影视。
- 豆瓣或 TMDB 降级时可使用已缓存元数据，无法形成可靠推荐时隐藏区块，不用随机内容填充。

### PRD-PLAY-007 广告时间段追踪与跳过

- 广告标记绑定来源、来源条目、标准剧集和媒体版本指纹，至少包含开始时间、结束时间、来源和置信度。
- 标记来源分为系统检测、用户本人标记、多人聚合和管理员确认；不同媒体版本之间不得直接复用时间段。
- 用户可以在播放时标记“广告开始”和“广告结束”，提交前可调整时间并确认；本人标记立即对本人可用。
- 只有已注册、已登录、账户有效且已接受当前声明的用户可以创建或提交广告标记；会话失效时保留已有跳过能力，但标记操作必须先重新登录。
- 播放进入已确认广告段时显示“跳过广告”操作；用户开启自动跳过后，仅对高置信度或管理员确认的时间段自动执行。
- 自动跳过默认关闭；每次自动跳过必须显示可撤销提示，并允许恢复到广告段开始位置。
- 片头、片尾、预告和正片内容不默认视为广告；重叠、过短、越界或争议标记不参与自动跳过。
- 跳过只改变播放器当前时间，不删除、重写或伪造 HLS 清单和媒体分片。
- 用户可报告错误标记；系统保留脱敏聚合证据和版本历史，不记录完整媒体 URL。

### PRD-PLAY-008 播放页来源选集

- 播放器下方展示用户当前可用的配置来源，来源集合仍是全局启用与用户选择的交集。
- 每个来源独立展示线路、健康状态和实际可用剧集；不得假设不同来源剧集数量和命名完全一致。
- 用户先选择来源，再从该来源返回的剧集列表中显式选集；切换会创建新的安全播放会话。
- 能映射到同一标准剧集时沿用观看进度；无法可靠映射时必须明确提示，不自动回退到第 1 集。

## 9. 收藏、历史与偏好

### PRD-LIB-001 收藏

- 收藏对象是标准影视条目，不是单个 CMS 来源。
- 同一用户对同一影视只能收藏一次。
- 用户可在卡片、详情和收藏页取消收藏。

### PRD-LIB-002 观看历史

- 历史列表每个标准影视只显示一条，展示最后观看剧集和进度。
- 系统内部保留逐集进度，继续观看使用最后活跃剧集。
- 首版每用户最多保留 500 个影视历史条目，不按时间自动过期。
- 用户可删除单条或清空全部；清空后继续播放会重新创建记录。

### PRD-LIB-003 用户偏好

跨设备同步：

- 自动连播，默认开启。
- 高置信度广告自动跳过，默认关闭。
- 集数排序，默认正序。
- 搜索历史记录，默认开启。
- 首页推荐区块显示，默认开启。
- 用户启用的资源源。
- 播放器音量只保存在当前设备，不跨设备同步。

## 10. 资源源提交与审核

### PRD-SOURCE-001 用户提交

- 登录用户可提交名称、CMS API 地址、可选详情地址和说明。
- 首版不接受用户声明为成人专用的源；明确成人源直接拒绝。
- 相同规范化地址已有提交或全局源时，展示现有状态，不创建重复记录。
- 用户可查看自己的提交并撤回尚未进入最终审核的提交。

### PRD-SOURCE-002 自动检测

- 系统先执行网络安全检查，再探测 API 格式、搜索、详情和剧集能力。
- 安全检查失败直接终止，不能由管理员在 UI 中绕过。
- 能力检测失败进入“需要修改”，用户可复制原提交创建新版本。
- 检测结果必须包含时间、能力项和脱敏错误摘要。

### PRD-SOURCE-003 管理员审核

- 管理员只能审核已通过安全检查的提交。
- 批准、拒绝均需填写或选择审核结论；拒绝必须有对用户可见原因。
- 批准创建全局资源源但默认处于“已批准未启用”，由管理员确认后启用。
- 提交记录不可删除，作为审计依据保留。

### PRD-SOURCE-004 全局源生命周期

- 管理员可启用、暂停、恢复或归档全局源。
- 暂停立即阻止新搜索和播放会话；现有播放会话按安全策略自然过期，不无限续期。
- 连续健康检查失败使源进入“降级”，但不会自动永久归档。
- 归档源不再对用户展示，历史中保留来源名称但不能继续创建播放会话。

## 11. 管理端

### PRD-ADMIN-001 概览

显示用户数、活跃会话、待审核提交、资源健康、Provider 状态、Worker/队列和近期安全事件摘要。

### PRD-ADMIN-002 用户与邀请

- 创建、复制和撤销邀请。
- 查看用户状态、角色、创建时间和最近登录时间。
- 禁用、恢复用户和撤销全部会话。
- 首版不允许删除用户；避免历史和审计失去主体。

### PRD-ADMIN-003 Provider 与系统

- 查看豆瓣/TMDB 配置状态、健康和最近错误摘要。
- 管理端不得显示完整 API Key。
- 查看构建版本、数据库、Redis、Worker 和存储状态。
- 审计日志只读，不能通过管理界面删除或修改。

## 12. 数据导入与导出

### PRD-DATA-001 V1 导入

- 用户上传本地 JSON，系统先预览可迁移、跳过和失败项目。
- 允许导入资源选择、资源提交候选、观看历史、进度、搜索历史和偏好。
- 自定义源转换为待检测提交，不自动启用。
- 密码哈希、代理认证、临时导航、直接媒体 URL 和未知字段不导入。
- 同一文件重复导入保持幂等。

### PRD-DATA-002 V2 导出

用户可以导出自己的：

- 个人资料中的非敏感字段。
- 偏好和资源选择。
- 收藏。
- 观看历史与标准化进度。
- 搜索历史。
- 自己的资源源提交及状态。

导出不包含密码摘要、会话、邀请 token、上游媒体 URL、其他用户数据、系统秘密和完整审计日志。

## 13. 非功能产品要求

- 桌面与移动浏览器均可完成邀请、搜索、播放和审核核心流程。
- 所有异步页面必须设计加载、空、部分成功、失败、无权限和重试状态。
- 用户可见错误不泄露内部 URL、密钥、堆栈和第三方响应正文。
- 管理员操作必须产生审计事件。
- Provider 或单个资源源故障不能阻止用户访问自己的历史与收藏。
- 原型必须覆盖本文件中的关键状态，不能只画成功页面。

## 14. 首版成功标准

1. 管理员可以从空系统邀请用户，并完成资源源审核。
2. 普通用户可以从首页或搜索找到标准影视并开始播放。
3. 任一 Provider 或部分 CMS 失败时，产品能够明确降级。
4. 用户可在另一设备继续上次剧集与进度。
5. 播放失败时用户能够理解原因并确认切换到匹配剧集。
6. V1 导入不会创建未审核源或暴露旧媒体直链。
7. 权限测试证明普通用户无法访问管理员或其他用户数据。

## 背景与目标

产品范围、权限和状态机已确认。本阶段将规则转成可评审的用户旅程、页面流转、低保真线框和 Figma 制作清单，在任何页面开发前消除入口、反馈、异常恢复和跨角色交接歧义。

## 范围

- 邀请开户、登录和声明。
- 首页、搜索、详情、播放和手动换源。
- 收藏、历史、偏好、来源选择和数据迁移。
- 用户提交来源、系统检测、管理员审核与启用。
- 管理员邀请、用户治理、Provider 和系统状态。
- 桌面与移动端关键布局、页面状态和交互说明。

## 非目标

- 不生成前后端工程代码或 API 契约。
- 不决定数据库字段和缓存策略。
- 不以仓库线框替代 Figma 可编辑原型。
- 不虚构品牌视觉、最终动效和精确设计令牌。

## 验收标准

- 每条首版核心任务都有入口、主路径、异常、恢复和结束条件。
- 每个受保护入口体现未登录、声明过期和无权限行为。
- 搜索、播放、审核和导入覆盖其业务状态机。
- 桌面和移动端均能完成邀请、搜索、播放与审核主流程。
- Figma 页面、Frame、组件变体、导出物和评审状态有统一清单。

## 风险

- 未连接 Figma 时只能提交仓库内低保真规格，不能声称完成可点击高保真原型。
- 上游内容和播放错误状态较多，Figma 需优先复用状态组件，避免页面副本失控。

## 当前进展

- 已完成产品流程、低保真线框、交互规格和 Figma 制作清单。
- 已完成“首页/搜索 → 详情 → 播放失败 → 用户确认换源”本地可点击原型，并根据首轮反馈优化至 v0.2。
- 已继续迭代至 v0.3，加入用于解释导航、加载、选择、错误和恢复状态的动效系统，以及减少动态效果降级。
- 根据新增需求完成 v0.4 整版重设计，加入播放页同类推荐，以及按媒体版本追踪的广告标记、手动/可信自动跳过和撤销流程。
- v0.5 将搜索改为图标触发的下滑面板，在播放页增加按配置来源选集，并明确广告标记仅限有效注册用户。
- v0.6 根据反馈把搜索改为图标触发的居中悬浮模态框，补齐焦点、Esc 和遮罩关闭。
- 2026-07-14 用户同意沿当前方向推进；搜索交互继续修订至 v0.6。最终视觉、Figma 可编辑文件及 PNG/PDF 导出仍待补充。
- Windows 界面连接和内置浏览器当前均不可用，尚未在 Figma 中建立文件，也未完成自动化视觉检查。
- 下一步先人工评审本地原型，再迁移到 Figma 并补齐邀请、来源审核和 V1 导入流程。

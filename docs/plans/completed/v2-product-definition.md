# V2 产品定义方案

## 状态

- 状态：已完成
- 日期：2026-07-13
- 输入：V1 系统审查与 V1→V2 功能去向矩阵
- 输出：产品逻辑、角色权限、信息架构、状态机和需求追踪矩阵

## 目标

- 将功能矩阵中的每一项转化为明确、可验收的 V2 产品规则。
- 锁定首版角色、权限、数据可见性和管理边界。
- 定义前台、播放器、个人中心和管理端的信息架构。
- 定义邀请、账户、搜索、资源审核、播放和数据导入状态机。
- 为后续用户流程和 Figma 原型提供无歧义输入。

## 产品原则

1. 私有部署、登录后使用，不提供匿名内容浏览。
2. 内容元数据与播放资源分离，用户围绕标准影视条目操作。
3. 普通用户不能直接添加可立即访问的上游地址。
4. 部分 Provider 或资源源失败不能拖垮整个产品。
5. 播放失败不静默自动换源，涉及来源变化时由用户确认。
6. 隐私、安全和可解释性优先于复制 V1 的便利实现。

## 已收敛的矩阵项目

- 成人内容与成人源：不进入首版。
- 搜索历史：服务端跨设备同步，可关闭和清空。
- PWA：不进入首版。
- 移动端：不做长按倍速，保留控制锁。
- 用户数据导出：包含本人业务数据，不包含秘密或媒体 URL。
- 推荐标签：首版仅系统标签。
- 播放失败：推荐候选源，用户确认后切换。

## 非目标

- 不生成高保真视觉设计。
- 不决定数据库字段、API wire format 或具体缓存 TTL。
- 不初始化 Vben、FastAPI 或 Docker 服务。
- 不访问真实 Provider、CMS 或媒体资源。

## 验收条件

- 功能矩阵不再包含待产品确认项。
- 每个首版功能均有角色、入口、核心规则、异常状态和验收要求。
- 所有管理动作和用户私有数据都有明确权限。
- 页面树可以直接用于用户流程与低保真原型。
- 状态机覆盖正常、失败、取消、过期、禁用和恢复路径。
- 需求追踪矩阵能从 V1 功能追踪到 V2 页面、规则和状态机。

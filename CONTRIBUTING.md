# 贡献指南

项目当前处于 V2 重构阶段。

## 基本流程

1. 阅读 `README.md`、`AGENTS.md` 和相关开发方案。
2. 跨模块功能先在 `docs/plans/active` 建立方案。
3. 保持修改聚焦，并为行为变化补充测试。
4. 运行相关格式、Lint、类型、测试和构建检查。
5. 更新当日 `docs/worklogs` 记录。

## 提交要求

提交信息应清晰描述目的，建议使用 `feat`、`fix`、`refactor`、`test`、`docs`、`build`、`ci` 等类型。不得提交真实环境变量、密钥、数据库备份、日志或用户数据。

## 旧版

`legacy/v1` 是历史归档。除非任务明确要求，否则不接受针对旧版的新功能或依赖升级。
